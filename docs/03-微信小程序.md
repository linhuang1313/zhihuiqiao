# 微信小程序规划文档

## 概述

将智慧桥现有 Web 应用扩展为微信小程序端，利用微信生态的天然获客优势（小程序搜索、朋友圈分享、公众号跳转）降低获客成本，同时借助微信登录、微信支付和模板消息等原生能力，大幅提升用户体验和转化率。

---

## 现有架构

当前平台为 Web SPA，技术栈为 React 18 + TypeScript + Vite + Tailwind CSS + shadcn/ui，前后端通过 REST API 通信，后端为 Express + Drizzle ORM + SQLite。已有完整的用户认证（JWT）、需求发布、老师匹配推送、套餐解锁、订单管理等功能。

---

## 功能详情

### 小程序特有功能规划

| 功能 | 说明 |
|---|---|
| **微信一键登录** | 调用 `wx.login()` 获取 code，后端换取 openid，与平台账号绑定 |
| **微信支付** | 调用 `wx.requestPayment()`，使用 JSAPI 支付（`wx_lite` channel） |
| **订阅消息推送** | 替代 Web 端 `notifications` 表，向用户推送新需求匹配、订单状态变更等订阅消息 |
| **地理位置** | 调用 `wx.getLocation()` 自动填充家长地址或老师服务区域 |
| **手机号快捷获取** | 使用微信「快速填写手机号」组件，降低注册摩擦 |
| **分享裂变** | 老师可分享个人主页卡片到朋友圈/好友，携带邀请参数 |

---

### 用户流程（小程序核心路径）

#### 家长端

1. 打开小程序 → 微信授权登录（首次绑定手机号）
2. 首页展示推荐老师列表（可筛选）
3. 查看老师卡片 → 解锁详细联系方式（微信支付购买套餐）
4. 发布需求 → 系统自动匹配 → 收到订阅消息通知
5. 管理订单 → 完成后评价老师

#### 老师端

1. 打开小程序 → 微信授权登录
2. 完善个人资料
3. 收到需求匹配订阅消息 → 进入需求大厅
4. 申请接单（文档 `05-老师接单.md`）
5. 管理订单和评价

---

### 发布流程

1. **注册小程序账号**：在微信公众平台（mp.weixin.qq.com）注册，类目选「教育 > 培训机构」，需营业执照
2. **申请微信支付**：关联微信商户号或通过服务商接入
3. **申请订阅消息模板**：在小程序后台申请所需消息模板 ID
4. **开发与测试**：微信开发者工具调试，真机预览
5. **提审**：上传代码包（限制 2MB 主包 + 分包），填写功能描述，提交审核（通常 1–3 个工作日）
6. **上线**：审核通过后「发布」即上线

---

### 与现有后端的对接方案

小程序与 Web 端共享同一套后端 API，无需重写后端。主要新增或扩展：

| 接口 | 变更说明 |
|---|---|
| `POST /api/auth/wechat-login` | 新增：接收微信 code，换取 openid，返回 JWT Token |
| `POST /api/auth/bind-phone` | 新增：绑定微信手机号到平台账号 |
| `POST /api/subscribe/send` | 新增：触发订阅消息推送（内部服务调用） |
| `GET /api/users/wechat-openid` | 新增：查询 openid 是否已绑定账号 |
| 支付 API | 扩展：支持 `wx_lite` channel（需微信小程序支付参数） |

## 技术方案

### 技术选型对比

| 维度 | Taro（京东） | uni-app（DCloud） | 原生小程序（WXML/WXS） |
|---|---|---|---|
| **开发语言** | React + TypeScript | Vue 3 / React | WXML + WXS + JavaScript |
| **与现有代码复用** | 高（同为 React，逻辑层可复用） | 低（Vue 语法差异大） | 低（需全部重写） |
| **多端支持** | 微信/支付宝/百度/H5/App | 微信/支付宝/百度/H5/App | 仅微信 |
| **组件库** | `@tarojs/components` + NutUI | uni-ui / uView | WeUI |
| **与 TypeScript 支持** | 原生支持 | 支持但配置繁琐 | 支持但生态有限 |
| **性能** | 良（虚拟 DOM 层额外开销） | 良 | 优（无运行时框架开销） |
| **社区生态** | 活跃 | 活跃 | 官方文档完善 |
| **上手成本** | 低（React 团队） | 中 | 中 |
| **包体积** | 较大（Taro 运行时约 80KB） | 较大 | 最小 |
| **适用场景** | React 团队，追求代码复用 | Vue 团队，多端 | 仅小程序，极致性能 |

**推荐方案：Taro 4 + React 18 + TypeScript**

理由：
1. 当前团队已熟悉 React + TypeScript，迁移成本最低
2. 业务逻辑（API 调用、状态管理、工具函数）可大量复用
3. Taro 4 原生支持 React 18 并发模式
4. 为未来扩展支付宝小程序或 App 保留空间

---

### 代码复用策略

#### 可直接复用（迁移成本极低）

| 类型 | 具体内容 |
|---|---|
| API 请求层 | `client/src/lib/queryClient.ts` 中的 fetch 封装（替换 URL 即可） |
| 类型定义 | `shared/schema.ts` 中所有 TypeScript 类型 |
| 业务逻辑 Hooks | `useAuth`, `useDemands`, `useTeachers` 等无 DOM 依赖的 Hooks |
| 工具函数 | 格式化、校验、计算等纯函数 |
| 状态管理 | Zustand store（Taro 兼容） |

#### 需要适配重写（成本中等）

| 类型 | 具体内容 | 原因 |
|---|---|---|
| UI 组件 | shadcn/ui 所有组件 | 依赖 DOM API，小程序不支持 |
| 路由 | wouter → Taro Router | 小程序有自己的路由栈 |
| 样式系统 | Tailwind CSS → `@taroify` 或自定义样式 | 小程序不支持部分 CSS 特性 |
| 图片上传 | `<input type="file">` → `wx.chooseImage` | 小程序原生 API |

#### 需要全新开发（小程序特有）

| 功能 | 说明 |
|---|---|
| 微信登录授权页 | `wx.login` + `getUserProfile` 流程 |
| 支付调起逻辑 | `wx.requestPayment` |
| 订阅消息申请 | 用户授权接收订阅消息的弹窗 |
| 分享卡片 | `onShareAppMessage` 配置 |
| TabBar 导航 | 小程序底部导航栏配置 |

**综合估算复用率：约 40–50% 业务逻辑可复用，UI 层需 80% 重写。**

## 安全考虑

- **Session 安全**：微信 `session_key` 只存后端，不传给前端；JWT Token 存 `wx.setStorageSync`
- **支付安全**：商户号私钥只在服务器侧，小程序不持有任何支付密钥
- **数据合规**：获取用户位置需在使用时申请，不在启动时一次性请求所有权限（微信审核要求）
- **网络安全**：小程序只允许 HTTPS 域名，需提前在微信后台配置合法域名白名单
