# 老师学历认证功能规格

## 概述

为老师提供学历材料上传入口，由管理员审核后颁发「已认证」标识，在老师列表和详情页醒目展示，增强家长对老师背景的信任感，提升平台解锁率和成单率。

---

## 现有架构

`teacher_profiles` 表已有 `education`（学校）、`major`（专业）、`degree`（学历等级）和 `verified`（布尔值）字段，但均为老师自填文字，无材料佐证，`verified` 字段目前仅能由管理员在后台手动翻转，缺乏材料关联和审核流程。

---

## 功能详情

### 用户流程

#### 老师端

1. 老师进入「个人资料」→「学历认证」tab
2. 查看当前认证状态（未认证 / 审核中 / 已认证 / 已拒绝）
3. 若未认证或已拒绝，可上传以下材料（至少上传 1 项）：
   - 学生证照片（在校生）
   - 学位证书照片（毕业生）
   - 学信网查询截图
   - 其他辅助材料（可选）
4. 填写补充说明（可选）
5. 提交审核，状态变为「审核中」，上传入口禁用
6. 收到审核结果通知（通过 / 拒绝 + 拒绝原因）
7. 若被拒绝，可重新修改材料后再次提交

#### 管理员端

1. 管理员进入后台「认证审核」列表
2. 查看「待审核」列表，按提交时间排序
3. 点击某条申请，查看老师基本信息 + 上传材料（图片预览）
4. 操作：
   - **通过**：`status = 'approved'`，`teacher_profiles.verified = true`，发送通知给老师
   - **拒绝**：填写拒绝原因，`status = 'rejected'`，发送通知给老师
5. 已处理记录可在「历史记录」中查询

### 数据库设计

#### 新增表：`teacher_certifications`

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER PK | 自增主键 |
| `teacher_id` | INTEGER NOT NULL | references `users.id`（老师用户 ID） |
| `material_type` | TEXT NOT NULL | 材料类型：`student_card` / `degree_cert` / `xuexin_screenshot` / `other` |
| `image_url` | TEXT NOT NULL | 文件存储 URL |
| `file_name` | TEXT | 原始文件名 |
| `file_size` | INTEGER | 文件大小（字节） |
| `status` | TEXT NOT NULL DEFAULT `'pending'` | `pending` / `approved` / `rejected` |
| `admin_note` | TEXT | 管理员审核备注（拒绝原因等） |
| `reviewed_by` | INTEGER | references `users.id`（审核管理员 ID） |
| `submitted_at` | INTEGER | 提交审核时间戳 |
| `reviewed_at` | INTEGER | 审核完成时间戳 |
| `created_at` | INTEGER | 记录创建时间戳 |

**说明**：一次提交可包含多条记录（每个材料文件一行），通过 `teacher_id` + `submitted_at` 分组为一次「申请批次」。

#### `teacher_profiles` 表变更

| 变更类型 | 字段 | 说明 |
|---|---|---|
| 新增 | `certification_status` | TEXT：`uncertified` / `pending` / `certified` / `rejected`（冗余字段，便于列表查询） |
| 新增 | `certified_at` | INTEGER：认证通过时间戳 |
| 已有 | `verified` | 认证通过时置 `true`，拒绝时置 `false` |

### API 设计

| Method | Path | 描述 |
|---|---|---|
| `POST` | `/api/teacher/certifications/submit` | 老师提交认证申请（含文件 URL 列表） |
| `GET` | `/api/teacher/certifications/status` | 老师查询自己的认证状态和材料列表 |
| `POST` | `/api/upload/certification` | 文件上传接口，返回存储 URL |
| `GET` | `/api/admin/certifications` | 管理员获取认证申请列表（支持 `status` 筛选） |
| `GET` | `/api/admin/certifications/:id` | 管理员查看单条申请详情 |
| `POST` | `/api/admin/certifications/:id/approve` | 管理员通过认证 |
| `POST` | `/api/admin/certifications/:id/reject` | 管理员拒绝认证（需填写 `reason`） |

**提交认证请求体：**

```typescript
{
  materials: [
    {
      materialType: 'student_card' | 'degree_cert' | 'xuexin_screenshot' | 'other',
      imageUrl: string,   // 先上传文件获得 URL，再提交
      fileName: string,
      fileSize: number
    }
  ],
  note?: string           // 补充说明
}
```

**文件上传响应体：**

```typescript
{
  url: string,    // 可访问的文件 URL
  key: string,    // OSS 对象 Key（用于后续删除）
  size: number
}
```

## 技术方案

### 认证状态流转

```
未认证 (uncertified)
    ↓ 提交材料
审核中 (pending)
    ↓ 管理员审核
已认证 (certified)  ←→  已拒绝 (rejected)
                              ↓ 老师修改后重新提交
                         审核中 (pending)
```

### 文件存储方案对比

| 维度 | 本地存储（服务器磁盘） | 阿里云 OSS | 腾讯云 COS |
|---|---|---|---|
| **成本** | 免费（受服务器容量限制） | 0.12元/GB/月，流量另计 | 0.118元/GB/月，流量另计 |
| **可靠性** | 低（单点故障，无冗余） | 高（99.9999999% 耐久性） | 高（同级别） |
| **扩展性** | 差（磁盘需人工扩容） | 无限扩展 | 无限扩展 |
| **访问速度** | 取决于服务器带宽 | CDN 全球加速 | CDN 全球加速 |
| **管理能力** | 弱 | 控制台/生命周期管理 | 控制台/生命周期管理 |
| **上线难度** | 极低 | 低（需注册账号） | 低 |
| **适用阶段** | MVP 测试 | 正式上线推荐 | 正式上线推荐 |

**推荐方案：阿里云 OSS**

认证材料属于用户隐私数据，需要高可靠性存储。推荐使用阿里云 OSS，并结合以下策略：
- 使用**服务端直传（STS Token）**模式：前端获取临时凭证后直接上传至 OSS，减少服务器带宽压力
- 图片存储路径：`certifications/{teacher_id}/{uuid}.{ext}`
- 访问控制：Bucket 设置为**私有**，通过后端生成时效性签名 URL（`signedUrl`，有效期 15 分钟）供前端预览
- 管理员审核完成后，可设置生命周期策略将已处理材料转移至低频存储降低成本

**文件限制：**

```typescript
const CERTIFICATION_UPLOAD_CONFIG = {
  maxFileSize: 10 * 1024 * 1024,  // 10MB
  allowedTypes: ['image/jpeg', 'image/png', 'image/webp'],
  maxFilesPerSubmission: 5,
};
```

## 安全考虑

- **访问权限**：认证材料图片 URL 仅限管理员和材料所有者访问，使用签名 URL 机制，不可直接猜测路径访问
- **文件内容校验**：后端验证 MIME 类型（不仅依赖扩展名），防止恶意文件上传
- **防刷**：同一老师审核中状态不可再次提交，已认证状态也不可重复提交
- **操作日志**：管理员每次审核操作记录 `reviewed_by` 和 `reviewed_at`，可追溯
