# 运营数据看板功能规格

## 概述

为管理员构建可视化运营数据仪表盘，将平台用户增长、需求转化、订单收入等核心业务指标以图表形式直观呈现，支持按时间段、城市、服务类别多维度下钻分析，辅助运营决策，并提供 CSV 数据导出能力满足线下分析需求。

---

## 现有架构

当前管理员后台已有基础的用户列表、订单列表、套餐管理等功能，均为表格形式展示原始数据，没有聚合统计视图。前端已安装 `recharts` 库（可直接使用）。数据库中已有 `users`、`demands`、`orders`、`user_purchases`、`payments`（文档 01 新增）等表，可作为看板数据来源。

## 功能详情

### 核心指标体系

#### 今日概览（数据卡片）

| 指标 | 数据来源 | 说明 |
|---|---|---|
| 今日新增用户 | `users.created_at` | 当日 0:00 至今注册用户数 |
| 今日新增需求 | `demands.created_at` | 当日发布的需求数 |
| 今日新增订单 | `orders.created_at` | 当日创建的订单数 |
| 今日 GMV | `payments.amount` WHERE `paid_at` 当日 | 当日实际支付金额总和 |
| 今日平台收入 | `orders.platform_fee` | 当日已付款订单平台佣金总和 |
| 活跃用户数 | 当日有操作记录（可从 session/log 中统计） | DAU |

#### 用户分析

| 指标 | 图表类型 | 说明 |
|---|---|---|
| 用户增长趋势 | 折线图 | 按日/周/月的新增用户数（家长/老师分线） |
| 用户角色分布 | 饼图 | 家长 vs 老师 vs 管理员占比 |
| 用户城市分布 | 柱状图（Top 10） | 各城市注册用户数 |
| 老师认证率 | 数据卡片 | 已认证老师 / 总老师数 |
| 用户状态分布 | 饼图 | active / frozen / banned 比例 |

#### 需求与转化漏斗

| 指标 | 图表类型 | 说明 |
|---|---|---|
| 需求发布趋势 | 折线图 | 按日/周/月的需求发布数 |
| 需求服务类别分布 | 饼图 | 音乐/体育/科目辅导等各类别占比 |
| 转化漏斗 | 漏斗图 | 需求发布 → 收到申请 → 匹配成功 → 下单 → 完成 |
| 需求状态分布 | 柱状图 | open / matched / closed / cancelled 各数量 |
| 平均成单周期 | 数据卡片 | 从需求发布到订单创建的平均天数 |

#### 订单分析

| 指标 | 图表类型 | 说明 |
|---|---|---|
| 订单量趋势 | 折线图 | 按日/周/月的订单数 |
| 订单状态分布 | 饼图 | scheduled / in_progress / completed / cancelled |
| 服务类别订单量 | 柱状图 | 各服务类别的订单数对比 |
| 平均订单金额 | 数据卡片 | `AVG(total_amount)` |

#### 收入分析

| 指标 | 图表类型 | 说明 |
|---|---|---|
| GMV 趋势 | 折线图（双轴） | GMV 总额 vs 平台佣金收入，按日/周/月 |
| 套餐购买收入 | 柱状图 | 各套餐的购买次数和总金额 |
| 支付渠道分布 | 饼图 | 微信支付 vs 支付宝占比（依赖文档 01） |
| 月度环比增长 | 数据卡片 | 本月 GMV 与上月对比，显示增减百分比 |

#### 老师分析

| 指标 | 图表类型 | 说明 |
|---|---|---|
| 活跃老师数趋势 | 折线图 | 按月统计有接单的老师数 |
| 老师评分分布 | 柱状图 | 1–5 分段的老师数量分布 |
| 老师接单量 Top 10 | 排行表 | 历史订单数最多的老师 |
| 老师服务类别分布 | 饼图 | 各技能类别的老师数量 |

### 数据库设计

看板模块**不新增表**，所有数据通过对现有表的聚合查询获得。为提升查询性能，需在以下字段上添加索引：

```sql
-- 时间范围查询优化
CREATE INDEX IF NOT EXISTS idx_users_created_at ON users(created_at);
CREATE INDEX IF NOT EXISTS idx_demands_created_at ON demands(created_at);
CREATE INDEX IF NOT EXISTS idx_orders_created_at ON orders(created_at);
CREATE INDEX IF NOT EXISTS idx_payments_paid_at ON payments(paid_at);

-- 多维度过滤优化
CREATE INDEX IF NOT EXISTS idx_users_city ON users(city);
CREATE INDEX IF NOT EXISTS idx_demands_service_category ON demands(service_category);
CREATE INDEX IF NOT EXISTS idx_orders_service_status ON orders(service_status);
```

| Method | Path | 描述 |
|---|---|---|
| `GET` | `/api/admin/stats/overview` | 今日概览数据卡片 |

| `GET` | `/api/admin/stats/orders` | 订单量与状态统计 |
| `GET` | `/api/admin/stats/revenue` | GMV 与收入统计 |

**通用查询参数：**

```
?startDate=2026-01-01
&endDate=2026-03-31
&granularity=day|week|month   // 时间粒度
&city=上海                      // 城市筛选（可选）
&serviceCategory=音乐陪伴       // 服务类别筛选（可选）
```

**用户增长接口响应示例：**

```typescript
{
  growth: [
    { date: '2026-03-01', parents: 12, teachers: 5, total: 17 },
    { date: '2026-03-02', parents: 8, teachers: 3, total: 11 },
    // ...
  ],
  roleDistribution: [
    { role: 'parent', count: 342 },
    { role: 'teacher', count: 89 },
  ],
  cityDistribution: [
    { city: '上海', count: 156 },
    { city: '北京', count: 98 },
    // ...
  ]
}
```

**GMV 趋势接口核心 SQL 示例：**

```sql
SELECT
  DATE(datetime(paid_at, 'unixepoch')) AS date,
  SUM(amount) AS gmv,
  COUNT(*) AS payment_count
FROM payments
WHERE status = 'paid'
  AND paid_at >= :startDate
  AND paid_at <= :endDate
GROUP BY date
ORDER BY date ASC;
```

**CSV 导出查询参数：**

```
?type=users|demands|orders|payments
&startDate=2026-01-01
&endDate=2026-03-31
```

## 技术方案

### Recharts 组件选型

| 图表类型 | Recharts 组件 |
|---|---|
| 折线图（趋势） | `<LineChart>` + `<Line>` |
| 折线图（双轴） | `<ComposedChart>` + `<Line>` + `<Bar>` + `<YAxis yAxisId>` |
| 饼图（分布） | `<PieChart>` + `<Pie>` + `<Cell>` |
| 柱状图（对比） | `<BarChart>` + `<Bar>` |
| 漏斗图 | `<FunnelChart>` + `<Funnel>` |
| 数据卡片 | 纯 React 组件（无需 Recharts） |

### 性能优化策略

- **后端聚合**：所有统计在 SQL 层完成，不在 Node.js 中遍历原始数据
- **缓存**：历史统计数据（非今日）缓存 1 小时（使用内存缓存或 Redis）
- **延迟加载**：各 Tab 的图表数据在切换到该 Tab 时才发起请求（懒加载）
- **请求合并**：「今日概览」Tab 的 6 个卡片在一个接口 `/api/admin/stats/overview` 中一次返回

### CSV 导出实现

```typescript
// server/routes/admin-export.ts
import { createObjectCsvStringifier } from 'csv-writer';

app.get('/api/admin/export/csv', requireAdmin, async (req, res) => {
  const { type, startDate, endDate } = req.query;
  const data = await getExportData(type, startDate, endDate);
  
  const csvStr = createCsvString(data, type);
  const filename = `wisdombridge_${type}_${startDate}_${endDate}.csv`;
  
  res.setHeader('Content-Type', 'text/csv; charset=utf-8');
  res.setHeader('Content-Disposition', `attachment; filename="${filename}"`);
  res.send('\uFEFF' + csvStr); // BOM 前缀，解决 Excel 中文乱码
});
```

## 安全考虑

- **权限验证**：所有 `/api/admin/stats/*` 和 `/api/admin/export/*` 接口必须验证 `role = 'admin'`，非管理员返回 403
- **数据脱敏**：统计接口只返回聚合数据（计数、求和、平均值），不包含任何用户个人信息（姓名、手机号等）
- **导出限制**：单次导出时间范围不超过 1 年，单次导出数据行数不超过 10 万条，防止服务器内存溢出
- **操作日志**：CSV 导出操作记录到管理员操作日志（谁在什么时间导出了什么数据）

## 优先级与依赖

- **优先级：P2** — 对运营决策有价值，但不影响核心用户功能，建议在其他 P0/P1 功能稳定后开发
- **前置依赖**：
  - 文档 `01-支付接入.md`：`payments` 表是收入分析的核心数据来源
  - 文档 `02-学历认证.md`：认证率指标依赖 `teacher_certifications` 表
  - 文档 `05-老师接单.md`：转化漏斗中的「收到申请」节点依赖 `demand_applications` 表
- **建议开发顺序**：01 → 02 → 05 → 06（看板作为最后一个功能，能覆盖最完整的数据维度）
- **后续迭代**：
  - 大盘异常告警（指标骤降时推送钉钉/企业微信通知）
  - 自定义报表（管理员自选指标组合）
  - 数据对比（同比/环比自动计算并高亮显示）
