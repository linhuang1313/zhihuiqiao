# 智慧桥 WisdomBridge — 项目总览

> 最后更新：2026-03-29

---

## 项目定位

大学生/研究生兼职家教撮合平台，连接家长（孩子2-18岁）和高学历兼职老师。

- **平台名称**：智慧桥（WisdomBridge）
- **理念**：智慧和知识的桥梁链接和传递
- **目标用户**：
  - 家长端：有 2-18 岁孩子的家长
  - 老师端：在校大学生、研究生等高学历兼职者
- **服务类别**：音乐陪伴、体育培训、科目辅导、兴趣培养、氛围陪伴、其他

## 已实现功能（MVP）

### 用户系统
- 三种角色：家长（parent）、老师（teacher）、管理员（admin）
- 登录/注册，session 认证
- 管理员可冻结/封禁用户

### 家长端
- 发布需求（服务类别、孩子年龄、预算、时间、地点）
- 浏览老师列表（脱敏显示，未解锁只能看到模糊信息）
- 购买解锁套餐 → 管理员确认 → 解锁老师完整信息
- 查看订单、评价老师

### 老师端
- 完善个人资料（学历、技能、服务区域、时间、价格）
- 通知中心（铃铛图标 + 未读数 + 通知列表）
- 工作台显示最新匹配推送
- 接单大厅、收入统计

### 管理员端
- 用户管理（查看/冻结/封禁）
- 老师认证审核
- 购买确认（手动确认家长的套餐购买）
- 收入面板（总收入、待确认、已确认）
- 订单管理

### 自动匹配推送
- 家长发布需求 → 系统自动计算匹配分数 → 通知 Top 5 已认证老师
- 匹配算法：技能 25分 + 地区 30分 + 价格 25分 + 时间 20分

### 付费模型
- 解锁套餐：家长购买套餐后可查看老师完整信息
- 订单抽佣：每笔订单平台抽 10%
- 老师信息脱敏：未解锁显示模糊姓名、隐藏手机号/头像/详细学历

## 商业模式

### 核心定位
平台卖的是「撮合效率」——家长省去大海捞针找人的时间，老师省去几小时浏览找兼职的时间。

### 创始人约束
- 没有融资，不自费补贴
- 不做增长烧钱，靠产品本身吸引用户

### 分阶段盈利路线

#### 第一阶段：验证需求（不收费）
- 目标：一个城市、一个品类，凑齐 10 个老师 + 5 个家长，手动撮合 3-5 单
- 老师端：免费入驻
- 家长端：免费使用
- 唯一目的：验证"家长愿意付钱给大学生做陪伴"是否成立

#### 第二阶段：从家长端收钱
- 解锁套餐：29-99元/月，家长购买后才能看到老师完整信息
- 订单抽佣：每笔订单抽 10%（按 200元/4小时的单算，约 20元/单）
- 前提：平台老师比小红书随便找的更靠谱（学历认证 + 评价 + 担保）

#### 第三阶段：从老师端收钱
- 认证费：一次性 19-49元（覆盖学历审核成本）
- 置顶/推荐：9.9元/天 或 49元/周
- Qclaw 订阅：自动猎单工具 29-99元/月
- 前提：老师在平台接到的收入 > 在平台的花费

### 不做的事
- 不在冷启动阶段收费
- 不两边同时收费（先从家长收，老师免费降低供给门槛）
- 不搞补贴

## 防跳单策略

- 老师信息脱敏（不解锁看不到联系方式）
- 平台担保交易（资金经平台，完成后结算）
- 信用体系（平台内评价积累，跳出平台失去信用资产）
- 后期：聊天系统限制在平台内沟通

## Qclaw 集成规划

### 架构
```
外部信息源（58同城、本地群、豆瓣等）
        ↓  Qclaw Skill 抓取 + 解析
        ↓
   Qclaw Agent（跑在老师/运营电脑上）
        ↓  调用平台 API 回传结构化数据
        ↓
   智慧桥后端（统一任务池）
        ↓  匹配算法 + 分发
        ↓
   老师端通知 / 需求大厅展示
```

### 角色分工
- **家长**：不需要装 Qclaw，只用网站/小程序发需求、选老师、付款
- **老师**：自愿安装 Qclaw 作为"智能猎单助手"
- **平台**：维护统一任务池，做匹配分发和审核

### Qclaw Skill 规划
| Skill | 功能 | 阶段 |
|-------|------|------|
| demand-crawler-58 | 从58同城抓取家教/陪伴类兼职 | Phase 2a |
| demand-crawler-douban | 从豆瓣本地组抓取需求 | Phase 2a |
| demand-parser | LLM 解析非结构化文本为标准字段 | Phase 2a |
| auto-matcher | 按老师偏好自动筛选匹配需求 | Phase 2b |
| auto-reply | 生成定制化申请话术代老师提交 | Phase 2b |

### 实施路线
| 阶段 | 内容 | 预估工时 |
|------|------|----------|
| Phase 1 | 平台内老师主动接单 | 47h |
| Phase 2a | 外部需求 API + 审核后台 + Crawler Skill | 40h |
| Phase 2b | 自动匹配 + 自动申请 Skill | 35h |

## 本地启动

```bash
git clone https://github.com/linhuang1313/zhihuiqiao.git
cd zhihuiqiao
npm install
npm run dev
```

浏览器打开 `http://localhost:5000`

### 生产构建
```bash
npm run build
npm start
```

> 注意：Windows 系统已通过 `cross-env` 兼容环境变量设置。
