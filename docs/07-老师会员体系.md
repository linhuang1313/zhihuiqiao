# 老师会员体系（免费版 vs 付费版）

## 概述

老师端采用 Freemium 模式：免费版保证基本可用性（能接到单），付费版提供效率加速（接更多单、更快接单）。目标是让免费老师成为平台"库存"吸引家长付费，同时通过付费版从活跃老师身上获得增量收入。

---

## 现有架构

当前老师端功能无分级，所有已认证老师享有完全相同的权限：
- 无限接收匹配推送通知（`notifications` 表，`type = 'match_demand'`）
- 无限浏览需求大厅（待实现，见 `05-老师接单.md`）
- 无限申请接单（待实现）
- 查看完整需求详情

本功能需要在此基础上引入会员等级，区分免费版和付费版的功能权限。

---

## 会员等级定义

| 维度 | 免费版 | 加速版（¥49/月） |
|------|--------|-----------------|
| 每日推送匹配需求数 | 3 条 | 不限 |
| 每日主动申请接单次数 | 2 次 | 不限 |
| 需求详情可见范围 | 基础信息（类别、区域、孩子年龄） | 完整信息（+预算范围、时间偏好、特殊要求） |
| 家长搜索排名 | 普通排序 | 优先展示（排名置顶） |
| 新需求通知速度 | 延迟 30 分钟推送 | 实时推送 |
| 申请时 AI 话术生成 | 不可用 | 可用（根据需求自动生成自我介绍） |
| 个人主页标识 | 无 | 「加速版」金色徽章 |
| 月度数据报告 | 不可用 | 可用（接单数、曝光数、匹配趋势） |

## 数据库设计

### 新增表：`teacher_subscriptions`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `user_id` | INTEGER NOT NULL UNIQUE | references `users.id`（老师） |
| `plan` | TEXT NOT NULL DEFAULT `'free'` | `free` / `pro` |
| `started_at` | INTEGER | 付费版生效时间戳 |
| `expires_at` | INTEGER | 付费版到期时间戳 |
| `auto_renew` | INTEGER (boolean) DEFAULT `false` | 是否自动续费 |
| `created_at` | INTEGER | 创建时间 |

### 新增表：`teacher_daily_usage`

用于跟踪免费版每日用量限制。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `user_id` | INTEGER NOT NULL | references `users.id` |
| `date` | TEXT NOT NULL | 日期字符串 `YYYY-MM-DD` |
| `push_count` | INTEGER NOT NULL DEFAULT `0` | 当日已推送需求数 |
| `apply_count` | INTEGER NOT NULL DEFAULT `0` | 当日已申请次数 |

**联合唯一约束**：`(user_id, date)`

### 修改现有表

`teacher_profiles` 新增字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `is_pro` | INTEGER (boolean) DEFAULT `false` | 是否付费版（冗余字段，方便查询排序） |
| `boost_score` | INTEGER DEFAULT `0` | 排名加权分数（付费版 +50） |

## API 设计

### 会员相关

| Method | Path | 描述 |
|--------|------|------|
| `GET` | `/api/teacher/subscription` | 查询当前会员状态（plan、到期时间、用量） |
| `POST` | `/api/teacher/subscription/upgrade` | 升级为付费版（创建待支付订单） |
| `GET` | `/api/teacher/daily-usage` | 查询今日剩余推送/申请次数 |

### 需要修改的现有 API

| API | 修改内容 |
|-----|----------|
| `POST /api/demands`（自动匹配推送部分） | 免费老师每日最多推送 3 条，超过不推送；付费老师不限 |
| `GET /api/demand-hall/:demandId` | 免费老师返回基础字段，付费老师返回完整字段 |
| `POST /api/demand-hall/:demandId/apply` | 免费老师校验每日 2 次限制 |
| `GET /api/teachers`（家长端） | 付费老师 `boost_score` +50，排序时优先展示 |

## 核心逻辑实现

### 推送限制（修改 `routes.ts` 自动匹配部分）

```typescript
// 在 POST /api/demands 的自动匹配通知循环中
for (const teacher of scoredTeachers) {
  const subscription = await storage.getTeacherSubscription(teacher.id);
  const isPro = subscription?.plan === 'pro' && subscription.expiresAt > Date.now();

  if (!isPro) {
    // 免费版：检查今日推送次数
    const usage = await storage.getDailyUsage(teacher.id, today);
    if (usage.pushCount >= 3) continue; // 跳过，不推送
    await storage.incrementPushCount(teacher.id, today);
  }

  await storage.createNotification({ ... });
}
```

### 申请限制（修改申请接单 API）

```typescript
// 在 POST /api/demand-hall/:demandId/apply 中
const subscription = await storage.getTeacherSubscription(req.session.userId);
const isPro = subscription?.plan === 'pro' && subscription.expiresAt > Date.now();

if (!isPro) {
  const usage = await storage.getDailyUsage(req.session.userId, today);
  if (usage.applyCount >= 2) {
    return res.status(429).json({
      message: '今日申请次数已用完，升级加速版可无限申请',
      upgradeUrl: '/teacher/membership'
    });
  }
  await storage.incrementApplyCount(req.session.userId, today);
}
```

### 需求详情脱敏（修改需求大厅详情 API）

```typescript
// 在 GET /api/demand-hall/:demandId 中
const isPro = await isTeacherPro(req.session.userId);

const response = {
  id: demand.id,
  serviceCategory: demand.serviceCategory,
  specificService: demand.specificService,
  childAge: demand.childAge,
  location: demand.city, // 免费版只到城市级别
  // 以下字段仅付费版可见
  ...(isPro ? {
    budgetMin: demand.budgetMin,
    budgetMax: demand.budgetMax,
    preferredTime: demand.preferredTime,
    specialRequirements: demand.specialRequirements,
    serviceType: demand.serviceType,
  } : {
    budgetMin: null,
    budgetMax: null,
    preferredTime: '升级加速版查看',
    specialRequirements: '升级加速版查看',
  })
};
```

### 排名加权（修改老师列表排序）

```typescript
// 在 GET /api/teachers 中
const teachers = await storage.getAllTeachers();
// 付费老师优先排序
teachers.sort((a, b) => {
  const aBoost = a.profile?.is_pro ? 50 : 0;
  const bBoost = b.profile?.is_pro ? 50 : 0;
  return (bBoost + (b.profile?.ratingAvg || 0) * 10)
       - (aBoost + (a.profile?.ratingAvg || 0) * 10);
});
```

## 升级触发点（转化漏斗）

在以下场景引导免费老师升级：

| 触发场景 | 展示方式 |
|----------|----------|
| 今日推送已满 3 条，有新需求未推送 | 通知中心显示"还有 X 条匹配需求，升级查看" |
| 今日申请已满 2 次 | 申请按钮变灰 + 弹窗"升级加速版，无限申请" |
| 查看需求详情时预算等字段被隐藏 | 模糊区域 + "升级查看完整信息" 按钮 |
| 老师工作台 | 常驻"升级加速版"卡片，展示付费版权益 |
| 每月月初 | 推送上月数据摘要 + "付费版老师平均多接 X 单" |

## 到期与续费

- 付费版按月计费，到期后自动降级为免费版
- 到期前 3 天发送续费提醒通知
- `auto_renew` 字段预留，待接入真实支付后启用
- 当前阶段：升级操作创建一笔待确认的购买记录（复用 `user_purchases` 表），管理员手动确认后生效
