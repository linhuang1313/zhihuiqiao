# 平台内聊天系统功能规格

## 概述

在智慧桥平台内建立家长与老师的即时通信通道，取代用户跳出到微信私聊的行为，将沟通过程留在平台内以便平台监控服务质量、保护双方权益，同时通过「已解锁或有订单才可发起聊天」的准入机制，进一步推动套餐购买转化。

---

## 现有架构

当前平台无内部消息系统。家长通过「解锁」获取老师手机号后，直接跳出到微信或电话沟通，平台无法追踪后续互动。`notifications` 表仅用于系统单向通知（匹配提醒、订单状态变更），不支持双向实时消息。

---

## 功能详情

### 用户流程

#### 发起聊天（家长视角）

1. 家长在老师详情页点击「发消息」按钮
2. 系统校验准入条件：
   - 已解锁该老师（`unlock_records` 存在对应记录），**或**
   - 双方有关联订单（`orders` 表中 `parent_id` + `teacher_id` 匹配）
3. 若不满足，提示「请先解锁老师资料」并引导购买套餐
4. 若满足，检查是否已有会话（`conversations` 表查询），有则直接进入，无则创建新会话
5. 进入聊天窗口，发送文字消息

#### 发起聊天（老师视角）

1. 老师在订单详情页或家长列表页点击「发消息」
2. 准入条件：存在订单关系（老师不能主动联系未解锁自己的家长）
3. 后续流程与家长端一致

#### 消息接收

1. 对方在线：WebSocket 实时推送，消息即时显示，无需刷新
2. 对方离线：消息保存入库，对方下次打开时加载；发送系统通知（`notifications` 表）告知有新消息

#### 系统消息（自动触发）

- 订单创建时：「[家长名] 已接受您的申请，查看订单详情」
- 订单支付时：「订单已付款，请准时提供服务」
- 订单完成时：「服务已完成，请对彼此进行评价」
- 系统消息只展示，不可回复

### 数据库设计

#### 新增表：`conversations`

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER PK | 自增主键 |
| `parent_id` | INTEGER NOT NULL | references `users.id` |
| `teacher_id` | INTEGER NOT NULL | references `users.id` |
| `last_message_id` | INTEGER | references `messages.id`（最新消息，用于列表展示） |
| `parent_unread_count` | INTEGER NOT NULL DEFAULT `0` | 家长未读消息数 |
| `teacher_unread_count` | INTEGER NOT NULL DEFAULT `0` | 老师未读消息数 |
| `status` | TEXT NOT NULL DEFAULT `'active'` | `active` / `archived` |
| `created_at` | INTEGER | 创建时间戳 |
| `updated_at` | INTEGER | 最后活跃时间戳（用于排序） |

**唯一约束**：`(parent_id, teacher_id)` 联合唯一，一对用户只有一个会话。

#### 新增表：`messages`

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | INTEGER PK | 自增主键 |
| `conversation_id` | INTEGER NOT NULL | references `conversations.id` |
| `sender_id` | INTEGER NOT NULL | references `users.id` |
| `type` | TEXT NOT NULL DEFAULT `'text'` | `text` / `image` / `system` |
| `content` | TEXT NOT NULL | 消息内容（文字内容或图片 URL） |
| `system_ref_type` | TEXT | 系统消息关联类型：`order_created` / `order_paid` / `order_completed` |
| `system_ref_id` | INTEGER | 系统消息关联 ID（如 `orders.id`） |
| `is_read` | INTEGER NOT NULL DEFAULT `0` | 接收方是否已读（0=未读，1=已读） |
| `created_at` | INTEGER | 发送时间戳 |

### API 设计

| Method | Path | 描述 |
|---|---|---|
| `POST` | `/api/conversations` | 创建或获取会话（幂等，若已存在返回现有会话） |
| `GET` | `/api/conversations` | 获取当前用户的会话列表（含最后一条消息和未读数） |
| `GET` | `/api/conversations/:id/messages` | 获取指定会话的历史消息（分页，按时间倒序） |
| `POST` | `/api/conversations/:id/messages` | 发送消息（文字或图片） |
| `PUT` | `/api/conversations/:id/read` | 标记会话消息为已读（清零未读计数） |
| `GET` | `/api/conversations/unread-count` | 获取当前用户总未读消息数（用于导航栏徽章） |
| `WS` | `/ws/chat` | WebSocket 连接入口 |

**WebSocket 消息协议：**

```typescript
// 客户端 → 服务端
{
  type: 'auth',
  token: string          // JWT Token，连接时发送
}
{
  type: 'message',
  conversationId: number,
  content: string,
  messageType: 'text' | 'image'
}
{
  type: 'read',
  conversationId: number
}

// 服务端 → 客户端
{
  type: 'message',
  message: Message,
  conversationId: number
}
{
  type: 'typing',
  conversationId: number,
  userId: number
}
{
  type: 'error',
  code: string,
  message: string
}
```

**分页参数（历史消息）：**

```
GET /api/conversations/:id/messages?before=<messageId>&limit=20
```

使用游标分页（`before` + `limit`），避免偏移分页在消息追加时产生重复或遗漏。

## 技术方案

### WebSocket 实现方案

基于 Node.js 原生 `ws` 库（Express 兼容），连接管理如下：

```typescript
// server/services/websocket.ts
import { WebSocketServer, WebSocket } from 'ws';

// 在线用户 Map: userId → WebSocket
const onlineUsers = new Map<number, WebSocket>();

export function setupWebSocket(server: HttpServer) {
  const wss = new WebSocketServer({ server, path: '/ws/chat' });

  wss.on('connection', (ws) => {
    let userId: number | null = null;

    ws.on('message', async (data) => {
      const msg = JSON.parse(data.toString());
      
      if (msg.type === 'auth') {
        userId = verifyJWT(msg.token);
        onlineUsers.set(userId, ws);
      }
      
      if (msg.type === 'message') {
        // 1. 校验准入
        // 2. 保存到数据库
        // 3. 推送给接收方（若在线）
        const receiver = getReceiverFromConversation(msg.conversationId, userId);
        const saved = await saveMessage(msg, userId);
        
        if (onlineUsers.has(receiver)) {
          onlineUsers.get(receiver)!.send(JSON.stringify({
            type: 'message',
            message: saved,
            conversationId: msg.conversationId
          }));
        } else {
          // 发送离线通知
          await createNotification(receiver, 'new_message', saved);
        }
      }
    });

    ws.on('close', () => {
      if (userId) onlineUsers.delete(userId);
    });
  });
}
```

### 消息保留策略

| 策略维度 | 规则 |
|---|---|
| **默认保留期** | 消息永久保留（SQLite 无自动清理） |
| **图片存储** | 图片上传至 OSS，消息只存 URL |
| **归档策略** | 超过 12 个月无互动的会话可标记 `archived`，不影响查询但不显示在列表默认视图 |
| **未来扩展** | 可引入定时任务清理 3 年以上消息，或提供用户手动清空会话的功能 |

## 安全考虑

### 防骚扰准入机制

```typescript
async function checkChatAccess(parentId: number, teacherId: number): Promise<boolean> {
  // 条件 1：家长已解锁该老师
  const unlock = await db.query.unlockRecords.findFirst({
    where: and(eq(unlockRecords.parentId, parentId), eq(unlockRecords.teacherId, teacherId))
  });
  if (unlock) return true;
  
  // 条件 2：双方有关联订单
  const order = await db.query.orders.findFirst({
    where: and(eq(orders.parentId, parentId), eq(orders.teacherId, teacherId))
  });
  return !!order;
}
```

### WebSocket 安全
- 连接建立后必须在 **5 秒内发送 auth 消息**，否则服务器主动断开
- 每条消息验证发送者属于该 `conversation_id` 的参与方
- 消息内容长度限制：文字 ≤ 1000 字符，图片 URL ≤ 2048 字符
- 频率限制：同一用户每秒最多发送 5 条消息，超出则断开连接

### 内容安全
- 文字消息可集成腾讯云内容安全 API（可选，未来迭代）
- 举报功能（未来迭代）：用户可举报消息，管理员审核
